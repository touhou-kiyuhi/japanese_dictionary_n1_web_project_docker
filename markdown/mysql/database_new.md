### 查詢第一筆資料
```sql
USE jlpt_n1_utf8mb4;
SELECT n1_w.word, n1_kr.kana, n1_kr.romaji, n1_w.meaning, n1_s.sentence, n1_s.sentence_meaning, n1_w.created_date, n1_w.last_modified_date
FROM n1_word n1_w
JOIN n1_kana_romaji n1_kr ON n1_w.id = n1_kr.word_id
LEFT JOIN n1_sentence n1_s ON n1_w.id = n1_s.word_id
WHERE n1_w.word = '亜';
```
### 檢查編碼與排序規則
```sql
USE jlpt_n1_utf8mb4; 
SHOW VARIABLES LIKE 'character_set%'; 
SHOW VARIABLES LIKE 'collation%';
```
* `SHOW VARIABLES LIKE 'character_set%';`: 查詢當前系統字元集設定
    * 作用：顯示 MySQL 中所有與 字元集（character set） 有關的系統變數。
    * 例如會看到：
        * `character_set_client`: 客戶端使用的字元集
        * `character_set_connection`: 連線使用的字元集
        * `character_set_database`: 資料庫的字元集
        * `character_set_server`: 伺服器預設字元集

    簡單說，就是用來確認資料庫、連線、表格等的編碼設定。
* `SHOW VARIABLES LIKE 'collation%';`: 查詢當前系統排序規則設定。
    * 作用：顯示 MySQL 中所有與 排序規則（collation） 有關的系統變數。
    * Collation 是指文字比較和排序的方法，像是大小寫敏感與否、日文或中文排序規則等。
    * 例如會看到：
        * `collation_connection`: 連線使用的排序規則
        * `collation_database`: 資料庫的排序規則
        * `collation_server`: 伺服器預設排序規則
